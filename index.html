「Webコンテンツ作成のテーマ」の成果成果

本テーマについて

手順書を読み実験を行ってレポートやプレゼンテーションにより報告するという実験の一連の流れを実践できるようになったり、モーション解析・編集ができるようになったり、HTML等を用いたWebページ作成ができるようになる。

プロジェクト課題について

私はプロジェクト課題のテーマにNHKの番組表を選びました。ここでは、ニュースやスポーツ、ドラマやアニメ等１２種類のジャンルとNHK総合テレビとEテレの２つのチャンネルから選べるようにテキストボックスを設置しました。検索ボタンを押すことでテキストボックスのジャンルやチャンネルに対応する番組名や番組開始・終了時刻などを表示するように設定しました。またどんなジャンルがあるのかわかりやすいように、テキストボックスの下にジャンルの一覧を表示させました。

プロジェクト課題で作成したページ: project.html

数当てゲーム

・乱数で 1〜10の値を1つ決めて、「答え」とする。変数yosoは常に4で固定する。正解ならば正解と、yosoが外れた場合はyosoよりも答えが大きいか小さいかをコンソールに出力されるようにする。

・予想のメッセージ「◯回目の予想: △」をコンソールではなくHTMLに出力するように書き換える。「◯」にあたる出力はspan#kaisu要素に、「△」にあたる出力はspan#answer要素に、出力先をp#result要素にする。

・テキストボックスに整数を指定して回答ボタンをつくり、押すと正解判定の結果が表示されるようにする。回答回数をカウントし、ページに表示させる。正解すると正解のメッセージを表示し、3回まちがうと答えを表示し、4回以上回答するとゲームが終わっているというメッセージを表示させる。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第１回WWWとHTML

今回の授業ではまずHTML要素について学習しました。そしてHTML要素を使ってHTML文書を書く学習をし、練習問題に取り組みました。その後、見出しの要素・リストの要素・ハイパーリンク・説明リスト要素・img要素を使って演習１−４に取り組みました。最後に今回学習したことを使って課題に取り組みました。

演習の成果: 演習1-4:HTMLのさまざまな要素

第２回CSSによるWebページの装飾

今回の授業ではWebページの装飾について学習しました。CSSに記述して文字の色や大きさを変えたり、リンクの色を変えたりマウスを重ねた時にテキストの装飾をなくす方法を学びました。最後に課題で前回の授業で作成したindex.htmlのCSSを作成してindex.htmlを装飾しました。

演習の成果: 練習問題2-3:ウェブブラウザ説明ページの装飾

第３回JavaScriptの基本

今回はJavaScript の基本について学習しました。まずJavaScriptでできることについて学びました。JavaScriptはWeb ページ上で複雑な機能を実現するプログラミング言語だということを知りました。またJavaScriptとjavaの違いについて学習しました。最後に課題で数当てゲームを作成しました。

演習の成果: 練習問題3-6:オブジェクトの参照

第４回DOMとJavaScript

今回はDOMとJavaScriptについて学びました。DOM と JavaScript を組み合わせることによって，Web コンテンツの内容を読み取ったり，コンテンツに変更を加えたりする処理の書き方を学びました。これを利用して数当てゲームを改良したり、プロジェクト課題のJavaScriptプログラムを改良してhtmlに表示させたりしました。

演習の成果: 演習4-2~4-5:DOMプログラミング

第５回イベント処理

今回はイベント処理について学びました。練習で様々なイベントを登録したときにどのような動作をするのかを学んだり、input要素でボタンを設置する方法を学んだりました。これらを使って課題で数当てゲームとプロジェクト課題にテキストボックスとボタンを設置しました。

演習の成果: 練習5-3: イベントを選択する

工夫点，苦心点，強調点，感想

  • 工夫点

    • 私が今回最も工夫した点はプロジェクト課題の検索結果の表示の仕方です。私は検索結果を見やすさ重視で表示したかったのでなるべくテレビの番組表のようにしようと思いました。まずどの行が何を表示しているのかわかりやすいように各行の上に何を表示しているのか表示させました。また表示結果をそれぞれ同じ横幅の枠で囲み見やすいように表示させました。他にもどんなジャンルがあるのかをわかりやすいように検索のテキストボックス・ボタンの下にジャンルの一覧を表示させました。

  • 苦心点

    • 私の今回の１番の苦心点はプロジェクト課題の検索結果の表示です。ただ検索結果を表示するだけでもプログラムが得意ではない私にとってとても大変だったのですが、とくに番組の開始時刻と終了時刻を「2022-03-04T04:50:00+09:00」このような形から「2022年03月04日04:50:00」のように直すのがとても難しかったです。また検索結果を見やすく表示するためにhtmlとJavaScriptとCSSの3種類のプログラムに変更を加えるのが大変でした。最初に数当てゲームでコンソールに表示していたものをHTMLに表示するようにJavaScriptに変更を加えるのが最初はよくわからずなかなかHTMLに表示できなくて苦労しました。

  • 強調点

    • index.htmlが見やすくなるようにCSSに変更を加えて見やすい色の組み合わせにしました。またHTMLの背景の黄色の濃さをちょうど良い濃さに調節しました。

  • 感想

    • HTML と JavaScript プログラムによる AJAX 通信を行う際にHTMLに「」を入れ忘れていることのなかなか気づけず、データを読み込むことができなく検索結果を表示できなくて大変でした。数当てゲームやプロジェクト課題でテキストボックス・ボタンを作成して動くようにするのはとても大変だったのっですが、その分完成した時の達成感がすごかったです。最後に今回のWebコンテンツ作成を通じて、HTML、CSS、JavaScriptの基礎を学び、それらを組み合わせて一つの機能を実現することの難しさと面白さを体感しました。特に、自分で工夫して作成した部分がうまく動作したときの喜びは非常に大きく、今後もプログラミングを続けていきたいという意欲が湧いてきました。
